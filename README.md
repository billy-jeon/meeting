# 정리록 — 회의 녹음 & 회의록 정리 웹앱

브라우저 마이크로 회의를 실시간으로 받아 적고, OpenAI API로 불렛포인트 회의록을
자동 정리해주는 정적(static) 웹앱입니다. 서버 없이 `index.html` 파일 하나로 동작하며
GitHub Pages에 그대로 배포할 수 있습니다.

## 주요 기능
- **녹음 시작 버튼** → PC 내장 마이크로 실시간 음성 인식 (브라우저 내장 Web Speech API 사용)
- **오른쪽 패널**: 실시간 스크립트 (인식되는 대로 즉시 표시)
- **왼쪽 패널**: 불렛포인트 회의록 초안 (일정 주기로 자동 정리 + 수동 "지금 정리" 가능)
- **하단 프롬프트 입력창**: "3번째 항목 더 구체적으로" 같은 지시로 회의록 초안 수정
- **.txt로 저장 버튼**: 최종 회의록을 텍스트 파일로 다운로드
- **OpenAI API 키**: 우측 상단 톱니바퀴(⚙)에서 입력 — 브라우저 메모리에만 보관되고
  새로고침하면 사라집니다 (저장/전송되지 않음)

## GitHub Pages 배포 방법
1. GitHub에 새 저장소 생성 (예: `meeting-notes`)
2. 이 폴더의 `index.html`을 저장소 루트에 업로드 (또는 git push)
3. 저장소 **Settings → Pages** 이동
4. **Source**를 `Deploy from a branch`로 설정, 브랜치는 `main`, 폴더는 `/ (root)` 선택 후 저장
5. 몇 분 후 `https://<사용자명>.github.io/<저장소명>/` 주소로 접속 가능

## 사용 전 꼭 확인할 점

**1. 브라우저 호환성**
실시간 음성 인식은 Web Speech API(`webkitSpeechRecognition`)를 사용하며,
현재 **Chrome(및 Chromium 계열, Edge)에서만 안정적으로 동작**합니다.
Safari/Firefox는 지원하지 않거나 동작이 불안정할 수 있습니다.

**2. 마이크 권한 & HTTPS**
GitHub Pages는 기본적으로 HTTPS로 서비스되므로 문제없지만, 브라우저가 마이크 접근을
요청하면 반드시 허용해야 합니다.

**3. OpenAI API 직접 호출과 CORS**
이 앱은 별도 백엔드 서버 없이 브라우저에서 곧바로 OpenAI Chat Completions API를
호출합니다. 대부분의 경우 정상 동작하지만, OpenAI 측 정책이나 네트워크 환경에 따라
브라우저 콘솔에 CORS 오류가 발생할 수 있습니다. 이 경우:
- 콘솔(F12)에서 정확한 오류 메시지를 확인하고,
- 필요하다면 Cloudflare Workers, Vercel Serverless Function 등 **경량 프록시 서버**를
  하나 앞단에 두고 `fetch` 주소만 프록시 주소로 바꾸면 해결됩니다.
  (이전에 진행하셨던 Airbridge 챗봇 프로젝트의 "GitHub Pages + Vercel 프록시" 구조를
  그대로 재사용하실 수 있습니다.)

**4. API 키 보안**
API 키를 브라우저에 직접 입력하는 구조이므로, 공용 PC나 여러 사람이 함께 쓰는
환경에서는 사용 후 브라우저 탭을 닫아 키가 메모리에서 사라지도록 하는 것이 안전합니다.
회사 내부용으로 여러 명이 계속 쓰실 계획이라면, 키를 서버(프록시)에 두고
프론트에서는 키 없이 프록시만 호출하는 구조로 바꾸는 것을 권장합니다.

## 커스터마이징 포인트
- `intervalSelect` 값으로 자동 정리 주기 조절 (1분 / 5분 / 10분 / 20분 / 수동 전용)
- `modelSelect`로 `gpt-4o-mini`(빠르고 저렴) / `gpt-4o`(품질 우선) 선택
- 회의록 정리 프롬프트는 `organizeNotes()` 함수 내 `sys` 변수에서 규칙 수정 가능
  (예: "결정사항 / 실행 항목 / 논의 내용"으로 3단 구분하고 싶다면 여기서 지시문 수정)

## Phase 2 — 오디오 원본 병행 녹음 (화자 구분 대비)
실시간 스크립트(Web Speech API)는 화자를 구분하지 못하므로, 녹음 버튼을 누르는 동안
`MediaRecorder`로 마이크 원본 오디오도 함께 캡처합니다. 하단 "오디오 원본 저장" 버튼으로
`.webm`(브라우저에 따라 `.ogg`/`.m4a`) 파일을 받을 수 있고, 이 파일을 이후 Phase 3에서
`gpt-4o-transcribe-diarize` 같은 화자 구분 모델에 업로드하면 화자 라벨이 붙은 정확한
버전으로 보강할 수 있습니다. (diarize 모델은 아직 실시간 스트리밍이 아니라 파일 업로드
방식만 지원하므로, 이번 단계에서는 "원본을 놓치지 않고 확보해두는 것"까지가 범위입니다.)
- 오디오는 10초 주기로 청크가 쌓이다가, 저장 시 하나의 파일로 합쳐집니다.
- 이 브라우저에서 `MediaRecorder`를 지원하지 않으면 자동으로 건너뛰고 실시간 스크립트만
  기록되며, 안내 토스트가 표시됩니다.
- 화자 구분 API 연동(Phase 3)은 아직 포함되어 있지 않습니다 — 지금은 원본 오디오 확보까지입니다.
